05 The language behind it

A hue per area

Indigo trips, teal people, sky routes, pink complaints, emerald payments. Colour teaches the app — never decoration for its own sake.

Rounded & warm type

A soft, curvy display face (SF Pro Rounded family) over a clean Avenir body — friendly and legible, the Zomato kind of warmth.

Painted, detailed icons

Two-tone glyphs sitting on irregular brush splats — defined and tactile, not flat outlines from a pack.

Quiet motion

Lists settle in with a gentle stagger; everything else stays still. Motion guides the eye, it doesn't perform.

How this gets built — without breaking anything

This is a visual direction only. When we implement, we change the presentation layer — layout, icons, colour, type — one screen per commit on a branch. We never touch data hooks, queries, mutations or navigation, so the app keeps working exactly as it does now. Anything that doesn't land reverts on its own; the rest stays. You run the real app to feel the true motion and data.
